## Purpose of this site

This site is a focused, equation-first environment for turbojet analysis. The goal is to keep the math, the solver, and the plots synchronized so changes in inputs have a transparent, traceable impact on the results.

### Why Cantwell-based analysis is used
- **Standardized reference frame:** Cantwell provides a widely accepted baseline for station definitions, matching logic, and cycle relationships.
- **Consistent symbols and tags:** Re-rendering the derivations preserves equation numbering and notation, which makes cross-checking easier.
- **Tag equations notice:** Equation tags match the numbers shown in Cantwell's in-text references.
- **Traceable assumptions:** The site mirrors the same assumptions used in the solver, so the analysis text and the output tables stay aligned.
- **Map and matching clarity:** The Cantwell flow makes it explicit how $f(M_2)$, $\pi_c$, and $\tau$ ratios relate, which supports the strip-model and operating-line plots.

### What each section contributes
- **Analysis:** Derivations and definitions used by the solver and plots.
- **Inputs + Tables:** A station-by-station snapshot of the MK1 solution used as the base state for all plots.
- **Performance + Diagnostics:** Visual trend summaries and matching constraints tied to the MK1 results.
- **Compressor Strip Model:** A simplified aerodynamic model that explains how compressor maps and matching lines behave.

### How to use the site
- Start in Analysis to align notation and assumptions.
- Run Inputs + Tables to establish a consistent base state.
- Use Performance + Diagnostics to interpret trends and constraints.
- Use the Strip Model to study map behavior and matching sensitivity.
- Treat plots as guidance for trends and consistency checks, not as a substitute for detailed component design.
